# SasyaVana Development Rules

## Rule 1

Never change database architecture without approval.

---

## Rule 2

Never generate code that bypasses business rules.

---

## Rule 3

Do not duplicate business logic.

---

## Rule 4

Backend validates everything.

Frontend never becomes the source of truth.

---

## Rule 5

Every feature follows:

Business

↓

Schema

↓

API

↓

Backend

↓

Frontend

↓

Testing

---

## Rule 6

Always consider:

- Security
- Scalability
- Offline support
- Localization
- Performance

---

## Rule 7

No temporary hacks.

No TODO implementations.

No placeholder business logic.

---

## Rule 8

Every module must be production ready.
